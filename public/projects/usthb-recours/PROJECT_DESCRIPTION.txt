TITRE DU PROJET
===============
USTHB Recours - Système de Gestion des Demandes de Changement de Spécialité


DESCRIPTION GÉNÉRALE
====================
USTHB Recours est une application web moderne et sécurisée développée pour 
digitaliser et automatiser le processus de demande de changement de spécialité 
. Le système fournit une interface intuitive pour les étudiants 
et un panneau d'administration complet pour le traitement efficace des demandes 
avec authentification, gestion des rôles, et automatisation des processus.


CONTEXTE & OBJECTIFS
====================
- Remplacer un processus manuel et inefficace par une solution digitale
- Simplifier la soumission des demandes de changement de spécialité
- Automatiser le traitement et le transfert des étudiants
- Centraliser la gestion administrative des demandes
- Améliorer la communication avec les étudiants via notifications automatiques
- Garantir la traçabilité et la sécurité des données


TECHNOLOGIES UTILISÉES
======================

Frontend & Framework:
  • Next.js 15.5.4 - Framework React full-stack avec SSR/SSG
  • React 19.1.0 - Librairie UI
  • TypeScript 5 - Langage typé pour meilleure qualité de code
  • Tailwind CSS 4 - Framework CSS moderne et responsive
  • Recharts 3.3.0 - Librairie pour graphiques et analytics

Backend & Infrastructure:
  • Next.js API Routes - Backend et microservices
  • Node.js/Express - Serveur backend
  • PostgreSQL (Supabase) - Base de données relationnelle
  • Redis (via ioredis) - Cache et rate limiting
  • JWT (jose) - Authentification par tokens

Sécurité & Authentification:
  • bcryptjs 3.0.3 - Hashage des mots de passe
  • JWT - Tokens d'authentification sécurisés
  • 2FA (Two-Factor Authentication) - Authentification à deux facteurs

Communications:
  • Nodemailer 7.0.6 - Envoi d'emails
  • Templates HTML - Emails responsives et professionnels

Déploiement:
  • Netlify - Hébergement et CI/CD
  • @netlify/plugin-nextjs - Intégration Next.js
  • netlify.toml - Configuration de déploiement


FONCTIONNALITÉS PRINCIPALES
=============================

1. PORTAIL ÉTUDIANT
   ✓ Soumission de demande de changement de spécialité
   ✓ Recherche automatique par matricule dans la base de données
   ✓ Auto-complétion des informations (nom, prénom, email, téléphone, spécialité)
   ✓ Validation en temps réel avec messages d'erreur contextuels
   ✓ Upload sécurisé de justificatifs (si nécessaire)
   ✓ Confirmation visuelle avec animations de succès
   ✓ Notifications email automatiques (confirmation, décision, transfert)

2. AUTHENTIFICATION & SÉCURITÉ
   ✓ Login sécurisé avec JWT et gestion de sessions
   ✓ 4 rôles d'accès (Viewer, Reviewer, Department Admin, Super Admin)
   ✓ 14 permissions granulaires pour contrôle d'accès fin
   ✓ Gestion des utilisateurs (création, modification, désactivation)
   ✓ Rate limiting - Maximum 5 tentatives de connexion/15 minutes
   ✓ Verrouillage automatique des comptes après tentatives échouées
   ✓ 2FA (Two-Factor Authentication) activé pour Super Admins
   ✓ Codes 2FA à 6 chiffres avec expiration de 10 minutes
   ✓ Appareils de confiance (bypass 2FA pendant 30 jours)
   ✓ Session timeout - Déconnexion après 30 min d'inactivité
   ✓ Rafraîchissement automatique du token JWT
   ✓ Password reset sécurisé avec tokens temporaires
   ✓ Logs d'audit complets pour traçabilité

3. PANNEAU D'ADMINISTRATION
   ✓ Dashboard centralisé avec statistiques en temps réel
   ✓ Graphiques des demandes (par jour/semaine/mois)
   ✓ Taux d'approbation/rejet en pourcentage
   ✓ Temps moyen de traitement des demandes
   ✓ Demandes par spécialité avec visualisation
   ✓ 4 cartes KPI (total, pending, approved, temps moyen)
   ✓ Filtrage avancé par statut, spécialité, date
   ✓ Recherche par nom/matricule avec résultats temps réel
   ✓ Tri des colonnes (date, nom, statut, spécialité)
   ✓ Vue détaillée de chaque demande avec historique
   ✓ Actions en masse (sélection multiple)
   ✓ Traitement rapide des demandes (approuver/rejeter)
   ✓ Aperçu des comptes verrouillés
   ✓ Gestion des appareils de confiance

4. AUTOMATISATION
   ✓ Transfert automatique de l'étudiant vers la nouvelle spécialité
   ✓ Mise à jour instantanée de la base de données
   ✓ Envoi d'emails automatiques selon la décision
   ✓ Validation instantanée des formulaires
   ✓ Mise à jour des statistiques en temps réel


ARCHITECTURE & STRUCTURE DU PROJET
===================================

Frontend Structure:
  • src/app/ - Pages et layouts Next.js
    - admin/ - Portail d'administration avec login, dashboard, gestion des demandes
    - api/ - Routes API backend
    - components/ - Composants réutilisables (forms, charts, headers)
  • src/lib/ - Utilitaires et logique métier
    - auth.ts - Logique d'authentification
    - supabase.ts - Client Supabase
    - email.ts - Templates et envoi d'emails
    - rateLimit.ts / rateLimitDB.ts - Gestion du rate limiting
    - twoFactor.ts - Logique 2FA
    - trustedDevices.ts - Gestion des appareils de confiance
    - permissions.ts - Système de permissions
    - passwordReset.ts - Logique de reset de mot de passe
  • src/hooks/ - Custom React hooks
    - useSessionTimeout.ts - Hook pour gestion du timeout de session

Backend Structure:
  • API Routes Next.js pour tous les endpoints
  • API endpoints:
    - /api/auth/* - Authentification (login, logout, refresh, 2FA, reset password)
    - /api/admin/* - Administration (gestion utilisateurs, demandes, statistiques)
    - /api/student/* - Endpoints étudiants
    - /api/submit-request/ - Soumission des demandes
    - /api/update-requests-table/ - Mise à jour des demandes
  • Middleware Next.js pour authentification et autorisation
  • Fonctions Netlify pour backends complexes
